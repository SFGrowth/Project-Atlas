## 1. Hypothesis Statement

> **H-FAE001:** Losing trades are not random. They occur under identifiable, measurable market conditions that can be detected before entry. If these failure signatures are statistically stable across time and execution models, Atlas can reduce drawdown and improve expectancy by systematically avoiding low-quality trades.

## 3. Research Design

### 3.1 Scope

The FAE will analyse every trade generated by Models A1, A2, and A3 across the full 2-year dataset (July 2024 – July 2026). The complete market state at entry will be reconstructed for each trade using approximately 40 features across five categories.

### 3.2 Feature Categories

| Category | Features |
|---|---|
| **Market Structure** | Swing high/low distance, trend maturity, pullback depth, breakout distance, range duration, compression/expansion state |
| **Trend** | ADX, ADX slope (5-bar), EMA21 alignment, EMA50 alignment, trend acceleration, trend exhaustion proxy |
| **Volatility** | ATR (14), ATR acceleration, ATR contraction ratio, volatility percentile (20-day) |
| **Liquidity/Session** | Distance to VWAP, prev day high/low distance, weekly high/low distance, relative volume, session, time of day, day of week, month |
| **Trade Context** | Consecutive wins/losses, daily P&L at entry, risk multiplier (ARI state), concurrent model exposure |

### 3.3 Analytical Pipeline

1. **Signal reconstruction** — regenerate all A1, A2, A3 signals with full feature capture
2. **Statistical contrast** — Cohen's d, Mann-Whitney U, t-test, Information Gain per feature
3. **Failure clustering** — K-Means + DBSCAN on losing trades only (data-driven clusters)
4. **Signature validation** — in-sample, out-of-sample, walk-forward, Monte Carlo, parameter sensitivity, cross-year stability
5. **Portfolio impact** — simulate filtered portfolios and estimate new PF, max DD, MC pass rate

---

## 4. Validation Requirements

A failure signature is eligible for promotion only if it satisfies **all** of the following:

| Criterion | Threshold |
|---|---|
| Sample size (losers in signature) | ≥ 15 trades |
| Effect size (Cohen's d) | ≥ 0.40 |
| Statistical significance | p < 0.05 |
| Information Gain | > 0 (positive) |
| Cross-year stability | PF improvement in ≥ 2 of 3 years |
| Trade count reduction | < 20% of total trades removed |
| Out-of-sample PF improvement | > 0 |
| Monte Carlo pass rate improvement | > 0 |

---

## 5. Candidate Outputs

Every validated failure signature will be classified as one of:

- **Execution Filter** — hard rule preventing entry when condition is present
- **Guardian Input** — soft rule reducing position size (ARI integration)
- **Observatory Monitor** — insufficient evidence; track for future re-evaluation

---

## 6. Risk Assessment

| Risk | Mitigation |
|---|---|
| Overfitting to in-sample data | Mandatory out-of-sample and walk-forward validation |
| Spurious correlations from small samples | Minimum N=15 per signature; Cohen's d threshold |
| Signature reduces trade count too aggressively | Hard cap: < 20% trade count reduction per signature |
| Black-box discovery | Every signature requires explicit market explanation |
| Interaction effects between signatures | Portfolio impact analysis must test combined filters |
